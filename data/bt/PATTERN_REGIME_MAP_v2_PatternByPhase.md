## 2. 各パターンの局面別「物語」

### 2-1. PatA = 大値幅期待（万能パターン）

**何を捉えるか**: H1 ATR が拡張中だが減速し始めた瞬間（RISING_DECEL）× vel3 8〜15 という「波の中盤の勢いある定常状態」。**H4 と H1 の方向が整合**しているのが必須条件。

**局面別物語**:
- **BU局面 × BUY (PF 2.32)** = 上昇拡張中の中盤順張り。H1とH4が揃って上 → 王道。
- **PD局面 × BUY (PF 1.87)** = 「PD = 収束中」でも、H1とH4が両方上向きなら、収束は「フェーズ移行」ではなく「一時的休憩」。中身のトレンドは生きている。
- **NONE局面 × BUY (PF 2.95)** = D1のATRクロス未確定下でも、H4×H1整合が取れていれば独立に機能する。**「中身の地合いがあればD1局面は無関係」**。
- **PD局面 × SELL (PF 1.78)** = 唯一機能する SELL。下降相場ではなく「上昇相場の押し」を売る逆方向ではない、**DN×PD のサンプルが多い**（N=18 PF 2.75）から効いている。
- **BU局面 × SELL (PF 0.75)** = BU=上昇拡張中に売る構造的逆風。

**構造的解釈**: PatA は条件が「H4/H1整合」と厳しいため、発火段階で**既に方向と勢いが揃っている**。局面ラベル（D1_Phase）が後から振られても、地合いを既に内包しているため局面の影響を受けにくい。これが**唯一の万能性**の根拠。

### 2-2. PatB = 押し目／戻り（PD一強）

**何を捉えるか**: H1 ATR が拡張中減速（RISING_DECEL）× vel3≥5 × **H1がH4と逆方向**。BUY なら「H4上昇中の H1 下押し → 押し目買い」、SELL なら「H4 下降中の H1 戻り → 戻り売り」。

**局面別物語**:
- **PD局面 × BUY (PF 2.45)** = 押し目買いの本領。PD（D1 収束）= 大局では拡張が一服しつつも、H1 では押し目を作っている瞬間。**「収束帯の中の押し」が綺麗に効く**。
- **BU局面 × BUY (PF 1.05)** = BU=拡張開始期に逆方向の押しを買う。「押し目過ぎ」状態が混在しており（v2マップで指摘済）、機能セルとボツセルが混在。
- **NONE局面 × BUY (PF 1.47)** = 中程度に機能。方向不定下では押し目買いの優位性が薄まる。
- **SELL側は全局面で機能しない** (PF 0.52〜0.83)。UP局面が 80% を占める中での戻り売り構造的逆風が出ている。**ただし DN×PD×MID-L のサブセルは PF 1.51 (N=9) と機能**するため、「DN局面が増えれば SELL も活きる可能性」は留保。

**構造的解釈**: PatB は「H1 が H4 と逆」という発火条件のため、本来的に「カウンタートレンド狙い」。これが効くのは「大局が落ち着き始めた PD局面」のみ。BU局面（立ち上がり）で逆方向の押しを買うと、立ち上がり力に押し戻されて死ぬ。

### 2-3. PatC = 初動（BU専用 / 局面選好極端）

**何を捉えるか**: H1 ATR が EXPANDING × ratio>1 × **H4=LOW × H1=MID/HIGH**。「H4 がまだ静かなうちに H1 で動き始めた」初動逆張り気味のエントリー。

**局面別物語**:
- **BU局面 × BUY (PF 3.46 WR 83.3%)** = **これだけが本物**。BU = 大局のATR拡張開始 → H4 が LOW で H1 が動き始める = 「**ボトムアウト→上昇開始**」の本質をピンポイントで捉える。
- **PD局面 × BUY (PF 0.48)** = PD = 収束中に「初動買い」を入れる = **方向不一致**。下方向の収束を逆張りで買うので死ぬ。
- **NONE局面 × BUY (PF 0.09 WR 12.5%)** = **最悪**。中立平坦＋拡張ピーク後の初動買い = **完全な高値掴み**。N=8 と小サンプルだがロスが圧倒的。
- **PD局面 × SELL (PF 2.05)** = SELL は逆に「PD=収束＝下方向への動き始め」と整合し、初動売りで機能する。
- **BU局面 × SELL (PF 0.50)** / **NONE × SELL (PF 0.19)** = BU では上昇開始に逆方向で売り、NONE では方向不定下で売り → 両方死亡。

**構造的解釈**: PatC は「初動逆張り気味」性質。**大局フェーズと発火方向の整合が絶対条件**。BU局面で BUY / PD局面で SELL のときだけ「初動 = フェーズ転換点の捕捉」になり機能する。それ以外は「フェーズ逆走」になり死ぬ。**局面選好が最も極端なパターン**。

### 2-4. PatD = H4 ATRクロス節目（節目順張り、PD大ボツ）

**何を捉えるか**: H4 ATR クロス後 3バー以内 × H1_ADX>18 × DI strength≥5。「H4 で大きな節目（ATR の方向転換）が来た直後に H1 で勢いを伴ってエントリー」。

**局面別物語**:
- **NONE局面 × BUY (PF 4.06 WR 68.8%)** = **NONE = D1 ATRクロス未確定** = 「H4 の節目が D1 より先行している」状況。H4 が先に動いて D1 が追従する段階で機能。
- **BU局面 × BUY (PF 1.37)** = BU = D1 もすでに拡張中、H4 節目順張りは普通に機能。
- **PD局面 × BUY (PF 0.68 N=50!!)** = **最大サンプルでの最大ボツ**。PD = D1 が収束に向かう中、H4 で押し目に対する逆方向の節目を捉えてしまう = **「D1 が下げに向かう中で H4 ATRクロスを使って買う」= 方向矛盾**。
  - Zone細分: MID-L (N=28 PF 0.85)、MID-H (N=18 PF 0.49) いずれもボツ寄り。**Zone を変えても救えない構造的ボツ**。
- **SELL側はN=10と極小**で評価困難だが、BU で機能 / PD で死亡の構造は同じ傾向（節目順張りの素直さ）。

**構造的解釈**: PatD は「H4 で先行する節目順張り」。D1 が NONE（未確定）の方が「H4 が D1 をリードする」関係が成立し機能。D1 が PD（収束中）だと「D1 全体は下げに向かうのに H4 で見つけた節目を使って買う」**逆走構造**になり、最大サンプルでの最大ボツが発生する。

→ **PatD × PD × BUY (N=50 PF 0.68 Net -163) は v2マップで未抽出だった重要新規ボツ候補**。

## 9. 次のアクション提案（カイ→メインおぱ）

### 9-1. 確度高い材料（即実装検討に値する）

1. **PatD × PD × BUY 除外フィルター追加**: N=50, PF 0.68, PF改善 +0.11単独。物語明確（節目順張りがPD方向と逆走）。
2. **PatC の局面別ロジック化**: PatC は局面で 0.09 〜 3.46 まで PF が振れる。**BUY なら BU局面のみ発火、SELL なら PD局面のみ発火**のラベル化で大幅改善可能。
3. **PatA 万能性の活用**: PatA は局面横断機能。**BUY は局面問わず通す**枠として安全に運用可能。

### 9-2. 追加分析候補（カイ実行可能）

- **DI Velocity × Pattern × Phase の4軸分析**: H1_DI_Plus_Vel3 等の動態指標で、特に PatD×PD×BUY の N=50 内に「実は機能するサブセル」が隠れていないか細分
- **時系列推移**: PatD×PD×BUY 50件が特定時期に集中しているか（局面継続中の連発か、散発か）
- **MFE/MAE 分析**: ボツセル群が「初期から逆走」か「TP近くで反転」かでフィルター精度向上の余地
- **DN局面追加サンプル取得**: 2023年データ追加BTで DN サンプルを倍増、PatA DN×PD×SELL の機能性確証へ

### 9-3. 戦略判断材料（カイは決定しない）

「ボツ削減主軸＋認識ツール思想」に照らすと:
- **PatD×PD×BUY のフィルター実装**は構造説明と数値根拠ともに明確 → 実装候補上位
- **PatC の局面別ラベル化**は「ラベル層と点数層を混ぜない」原則に整合（点数化でなく状態判定）
- **PatA の局面横断機能**は heatmap_v14 等の認識ツールで「安全パターン」として可視化候補
