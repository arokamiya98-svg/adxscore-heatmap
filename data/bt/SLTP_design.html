SL Design · ATR Average Base
// SL基準: 現在ATRではなく28本平均ATRを使用
ATR_Avg = AVERAGE( ATR[shift], ATR[shift+1], ... ATR[shift+27] )
 
SL_dist = ATR_Avg × 2.0 // デフォルト倍率
SL_price = EntryPrice ± SL_dist
LOWゾーンでは現在ATRが圧縮されているため、そのままSLに使うと構造的に狭すぎて則切りの連続になる。
28本平均ATRは「通常時のボラティリティ水準」を表し、LOWゾーン中でも適正なSL幅を確保できる。
TP Design · RR 1:1.6
// TP: SLを基準にRR倍率で設定
TP_dist = SL_dist × 1.6 // RR 1:1.6
TP_price = EntryPrice ± TP_dist
 
// 実測SL・TP幅（参考値）
SL_Pips avg ≈ 65〜80 pips
TP_Pips avg ≈ 104〜128 pips
▸ なぜRR 1:1.6か

LOWゾーンはトレンド前の蓄積期。爆発的なRRを狙うより、勝率と期待値のバランスを優先。RR 1:2だとTPが遠すぎてレンジで往復される。

▸ ATR平均を使う理由

LOWゾーン = ATRが中央値の70%以下に圧縮された状態。現在ATRをSLにすると則切りが構造的に発生する。平均で通常水準を参照することで解決。

▸ v1との違い

v1: 現在ATR × 1.2 → 狭すぎ（avg 78pips SL）
v3: ATR_Avg × 2.0 → 適正幅でパフォーマンス大幅改善

▸ 注意点

レンジ相場で捕まるシーンは構造上避けられない。トレンド初動を取れた時の利益でカバーする設計。勝率50%前後での期待値プラスを目指す。
