トレードツール（heatmap_v14, Scriptable widget, その他認識ツール）の設計時、本来の機能精度とは別に **「軽い演出」を1要素入れる**。例: 動的ステータスメッセージ（お店風 "冷やし中華始めました" 等）、絵文字、ラフな表現。

**Why:** あろさん明示（2026-06-05 Scriptable ATR Widget 実装中）：「**ラフな方がトレードモード突入しなくて済む**」。緊張感を持続するとメンタル消耗するため、ラフな表示が緩衝材として働く。これは [[mental-performance-research-track]] と [[roadmap-sensory-to-logic-phase]] の交差点：認識ツール自体に「内側=メンタル維持」の機能を組み込む発想。3層構造（外側ロジック化 / 中間認識ツール / 内側メンタル）のうち、中間と内側が結合する設計指針。

**How to apply:** 新規ツール設計時、データの一貫性・精度は崩さずに、視覚要素やラベル文言に「軽さ」を加える。ふざけすぎる前に「これでトレードモードを和らげられるか」を基準にする。具体例:

- Scriptable ATR Widget (2026-06-05) — オールグリーン時 "🍜 冷やし中華始めました"、混在時 "♨️ お湯沸いてます"、加熱時 "🔥 仕込み真っ最中"、回数超過 "🏪 本日は店じまい"
- heatmap_v14 — 配色・絵文字アイコン（FibTZ予測、D1 Phase、適正スコア D ≪NEUTRAL 等）

**注意**: 数値・状態判定そのものは絶対にラフにしない（認識の一貫性が損なわれる）。あくまで「表現」のレイヤーに軽さを入れる。
