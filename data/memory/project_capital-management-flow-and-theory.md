2026-06-01に本人言語化された資金管理ルール。**意思の有限性を外部構造（ルール+ボーナスシステム）でカバーする**設計。

## 重要な前提訂正

- **MT5レポートの数字（PF 0.73, Net -67,215 など）はクレジット込み計算** → 実態を反映しない
- **真の資金推移は「入出金履歴」基準** → わずかなプラスを常時維持してる
- 「Largest Loss -41,155」もクレジット込みの可能性 → 実損ではない可能性
- 「立て直し入金」と仮定したものは **ルール通りの計画的入金（サイクルの一部）**

## コア設計

### 1. リスク設計

- **3%は将来目標値**、現在は**さらに小さいリスク最小運用**で「増やせるか」を検証中
- 現在は初期状態 → 0.01ロット運用
- **SL想定: 最低600 pips を保持できる資金設計**
- 例: 7,000円入金 + 20%ボーナス = 8,400円資金枠 → 600pips SL対応可

### 1.1 連敗時の入金抑制ルール（核心原則）

- **連敗時の入金は「最小サイクル分」のみ** （3%相当 or 0.01ロットに対応できる分）
- 「**そこ崩したら、連敗でもDDが少ない資金管理の構造が全部無駄になる**」（本人言）
- = **入金額そのものが、トレード可能リスクの物理的上限**
- 負け→取り返したい→ロット上げる、という感情の連鎖を物理的に断ち切る装置

### 2. RR設定

- **1:1.6 〜 1:2** を狙う
- 勝率 50% でも期待値プラスになる設計

### 3. 2連勝1サイクル戦略（コア！）

```
[1st トレード] 0.01ロット → 勝ち
        ↓
[2nd トレード] 勝ち金額を乗せて 0.03ロット → もう一勝負
        ↓
[2連勝達成] → 出金 = 1サイクル完了
```

- トレンドフォローと「勝敗にも波がある」前提の戦略
- 連勝重なりで一気に資金回収
- 「コツコツドカン」回避、「コツコツ × ガツン回収」

### 4. プール機構

- 勝ち分の一部を HFM マイウォレットにプール
- 負けた時、そこから取引口座へ補充
- 効果:
  - トレード資金とトレード口座を切り分け
  - ボーナス（20%）を活かす
  - 連敗時もDDを抑制

### 5. ナンピン抑制構造

- 1トレードからのナンピン禁止 = **構造的に**抑制
- **「ひと波に対する1トレード」** の原則
- 入金タイミングを制約 → 感情によるロットアップを物理的に防ぐ
- 結果: 「コツコツドカン」できない構造

## 全体フロー

```
[HFM マイウォレット] (プール口座)
       │  ↓ 入金 (+20%クレジット)
       ↓
[取引口座] (1サイクル運用)
       │  
       ├─ 1st: 0.01ロット → 勝ち
       ├─ 2nd: 0.03ロット (乗せ) → 勝ち
       │       ↓ 2連勝達成
       │
       └─ 出金 → マイウォレットへ
            ↓ プール（次サイクル原資 + 連敗時バッファ）
```

## 軸の整理（重要）

**この設計は『資金計画アプローチ』が主、メンタル効果は副産物**。
あろさん本人の言葉: 「これは資金計画アプローチの共有ね！」

```
[1次目的] 資金計画ロジック    ← 設計の中心
[結果]    メンタル安定          ← 副産物として現れる
```

混ぜない。資金計画として論理的に組んだ結果、たまたまメンタル装置としても機能してるという順序。

## 資金計画アプローチの構成要素

1. **複利運用アプローチ**: 2連勝サイクルで勝ち分を次のロットに乗せる（0.01→0.03）
2. **トレンド相場との相性**: 連勝発生率が高い相場で最大効率
3. **構造的安全装置**:
   - コツコツドカン回避（ナンピン物理禁止）
   - 連敗ダメージ最小化（入金額が物理上限）
4. **利確の構造判断**: 2連勝達成=出金。「もっと取れた」を判断材料にしない
5. **「2連勝サイクル」が最も利益になる仕組み**: 単発の最大利益より、サイクルの再現性を優先
6. **副次効果としてのメンタル安定**: 上記が結果として「ぶれない遂行性」を生む

## 副次効果（メンタルへの影響）

資金計画ロジックの設計上、結果として以下が達成される:

- 意思力に頼らない判断（ナンピン・ロット上げ・損切躊躇が物理的に発生しない）
- 連敗時の感情的判断の遮断（入金上限が固定）
- 利確の自動化（サイクル完了=出金）
- 段階的成功体験の積み上げ（3%目標を将来に置いて、今はもっと小さく）

これらは [[external-mental-partner-role]] の「意思+外部装置のハイブリッド」と同じ構造を、**メンタル目的じゃなく資金計画目的で実装した結果**として持っている。

## クレジット系の正しい扱い

- TOP-UP BONUS 20% = **取引枠拡大の道具**（実弾ではない）
- ボーナスがあるおかげで、3%リスク × 600pips SL が小資金で成立
- MT5レポート上の数字は「クレジット込み」なので、実勝負での評価には使えない
- **実態判断は「入出金履歴」基準**

## 観察された運用例（2026年）

- 5/27: 出金 -20,880 → 入金 +20,931 → **100%ボーナス +20,931** (HFM一回限定イベント、自己資金20,931円→2倍化)
- 5/29: **イベント活用の戦術的ロットアップ** — 100%ボーナスで枠拡大した41,862円相当を0.05ロットでトレード。RISING_DECELで負け。自己資金リスクは7,000円相当（3%ルール内）。
- 5/30: +7,000入金 + 20%通常ボーナス（=ルール通りの新サイクル開始）

**重要な解釈**: 5/29のロットアップは **「感情でロット上げた」ではなく「構造で枠が広がったから利用」**。3%リスクルールは完全維持。負けは「メンタル管理対象」ではなく「確率管理対象」。

## イベントボーナス活用ルール（5/29事例から判明）

- HFMの**100%ボーナス**は一回限り（イベント時のみ）
- このタイミングで**枠を最大化してロットアップ**する戦術判断OK
- ただし**自己資金リスクは3%ルール維持**（枠が広がっても元手のリスクは変えない）
- 通常時の20%ボーナスとは別の運用判断

## HFM口座の履歴制約

- HFMは途中から使い始めたブローカー → MT5レポートは部分履歴のみ
- **全体収支の判断は「入出金履歴」基準で「微プラス」維持**
- HFMより前の他ブローカー履歴は別管理
