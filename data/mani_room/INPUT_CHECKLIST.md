## 💬 コメント欄 3つ（目的軸）

### ✏️ 新規理由 = 認識ロジック層（状態遷移）

**書く内容**:
- ATR / ADX の展開状況
- BU期 / PD期 / 凪 等のフェーズ
- 自分自身へのエントリー精度アンカー

**例**:
```
PD期、H4_T2、ATR収束底
ADX 25→30、DI+優位、押し目で粘る
```

> 💡 シグナルに乗ったわけじゃなくても、シグナルと似た状態遷移を書く。
> 「エントリー数値だけじゃ見れない情報」を残す。

---

### 🌍 決済理由 = マクロ環境層（バックグラウンド）

**書く内容**:
- 経済イベント / 地政学 / リスクオン・オフ
- **空欄OK、書きたい時だけ**
- あとで「なんでこんな値動きだったか」の説明素材

**粒度3段階**:

| レベル | 例 |
|---|---|
| 軽量 | `FOMC前` / `NFP当日` / `ホルムズ封鎖中` |
| 標準 | `FOMC前2日、ボラ薄、米株もみ合い` |
| ガッツリ | `ウクライナ侵攻3日目、リスクオフ継続、DXY強い、米株安、金買い圧` |

> 💡 ハッシュタグ風に書いてもOK（強制しない）。マニが文脈解釈で群クラスタリング。

---

### 📝 考察 = フィードバック層（結果ラベル）

**書く内容**:
- BUした / 凪離脱回避 / レンジ入り 等の結果ラベル
- あとから見た動きへの一言
- **軽め**

**例**:
```
BUに繋がった、凪離脱回避成功
```

## 🚫 反省欄

- **基本使わない**（空欄推奨）
- 書きたくなったら自由記述（強制しない）
- `#judge◎○△×` 等の判断質ラベルは廃止

> 💡 個別1トレードの judge は不確実性の世界では責任持てない。マニは群分析で評価する。

## 🌟 ラフ枠思想（最重要）

```
✅ 全部書く必要なし
✅ 書きたいとこだけ、書ける日だけ
✅ 数値だけ、絵文字だけもOK
✅ トレードモード突入を抑制（ラフ味キープ）
```

> 💡 きっちり書こうとするとトレードモードに突入して疲れる。
> マニはマニで群統計から拾える。完璧主義禁止。

## 📲 入力フロー（理想）

```
1. エントリー時
   ├ アプリ起動
   ├ 約定日時 / 価格 / オーダー / ロット 入力
   ├ ★（時間帯）選択
   ├ スタイル選択
   ├ タグ1個選択
   └ 新規理由テキスト記入（状態遷移）

2. エグジット時
   ├ 決済日時 / 価格 入力
   ├ 損益（相殺後値）入力
   └ 決済理由（マクロ環境メモ、書きたい時のみ）

3. あとで（夜 or 翌朝）
   └ 考察記入（フィードバック）
```
