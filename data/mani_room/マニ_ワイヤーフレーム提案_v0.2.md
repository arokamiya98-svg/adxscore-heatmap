## 1. レーン比率 最終確定

### マニ最終提案

```
セル全高を 100% として:
  日付ヘッダ           :   8%
  レーン1 (市場環境)    :  70%   ← 主役（v0.1 の 55% から拡大）
  レーン2 (シグナル品質):   8%
  レーン3 (結果)        :  14%
```

### 拡大の根拠

1. あろさん指示「環境レーンはもっと大きく」に応える
2. **HOT系ラベル除去で文字情報量が減る** → 代わりに**指標を縦に3〜4段積める**スペースが必要（案C 思想の取り込み先）
3. シグナル/結果は「**事実発生の記録**」だけなので、最小限のマーカー1個と数字1〜2行で足りる
4. 「環境を読みに来る」のがこのカレンダーの第一義 → 視覚面積でそれを保証

### レーン1 内訳（70%の使い方）

```
レーン1 (市場環境) 70% の内訳:
  D1 大局フェーズ       : 12%  ← BU/PD/凪 等の構造ラベル（薄いバッジ）
  H4 ADX 強度           : 22%  ← 背景濃淡 + ADX数値
  DI バランス           : 14%  ← 数値（DI+ - DI-）+ 矢印
  H1 ADX 強度           : 22%  ← 背景濃淡 + ADX数値
```

→ **案C の「縦軸=指標」をセル内部で再現**。1セル = 縦に積まれた小さな heatmap。

## 2. HOT系評価語の完全除去 — 代替表現

### 削除対象

- ❌ 「凪 / 弱 / 中 / HOT / 強」のテキストラベル
- ❌ それに準ずるカテゴリ語（warm / cold / mild など）
- ❌ 色名カテゴリ（red zone / green zone など）

### 代替表現（3層で重ねる）

#### 層1: ADX数値そのもの（事実の数字）

```
18  22  31  42  38
```

→ カテゴリではなく**生の数値**。読み手の経験に判断を委ねる。

#### 層2: 背景濃淡（強度の連続表現）

```
ADX値 → 不透明度 にマップ
0%(透明) ←──────────────→ 100%(濃)

例: ADX 18 → 薄い背景
    ADX 42 → 濃い背景
```

→ 段階分割せず**連続グラデーション**。閾値という概念をUIに出さない。

#### 層3: 色相（DI方向の偏り）

```
DI+ 優勢  → 青系
DI- 優勢  → 赤系
拮抗      → グレー系
```

→ **方向の偏り**を色相で。**強さは色相に含めない**（強さは層2の濃淡だけが担当）。

### 「偏りが強い日」がラベルなしで伝わるか

検証思考実験:

```
[Mon] ADX 18 / DI ⇄ +2   → 薄いグレー
[Tue] ADX 22 / DI ↑ +7   → 薄い青
[Wed] ADX 31 / DI ↑ +12  → 中間の青
[Thu] ADX 42 / DI ↑ +18  → 濃い青  ← ★ 一目でわかる
[Fri] ADX 38 / DI ↑ +9   → やや濃い青
```

→ **Thu のセルだけが視覚的に「濃く・色が強く・数字が大きい」三重で目立つ**。HOT というラベルを書かなくても、「今日は偏りが強い日」が地形として読める。

→ **判定: 伝わる**。むしろカテゴリラベルが消えたことで、「ADX 35 ≒ HOT」という閾値設計がUIに出なくなり、**エントリーロジック混入リスクが大きく下がる**（後述セルフチェック）。

### ボーナス: 「強さの質感」を出す微表現（オプション）

ラベルを使わずに「偏りの質」を出したい場合の補助手段:

| 表現 | 用途 | コスト |
|---|---|---|
| セル枠の太さ | ADX × DI スプレッド合成値で太く | 低 |
| セル右上の小さな縦バー | H1 ADX の日次内変動の幅 | 中（H1足ADX時系列が必要）|
| 数字のフォントウエイト | ADX値そのものの太さを変える | 低 |

→ **デフォルトでは使わない**。背景濃淡 + 色相 + 数値の3層で足りる。必要が出たら追加検討。

## 3. 別タブ：案C型 長尺ヒートマップ 日次版

### コンセプト

```
本タブ:  日次研究カレンダー（案B 改良版） … 月単位で「今日」をセルで読む
別タブ:  日次長尺ヒートマップ（案C 日次版） … 連続日次で「指標の地形」を読む
```

heatmap_v14 が**週次のマクロビュー**だったのに対し、別タブは**日次の連続ビュー**。
**同じ思想（縦軸=指標、横軸=時間）を粒度だけ日次に落とす**。

### タブ切替 UI 案（3パターン）

#### 案T1: タブ式（推奨）

```
┌─────────────────────────────────────────────────────┐
│ [📅 カレンダー本]  [📊 長尺ヒートマップ]            │
│ ─────────────  ─────────────────                    │
│                                                     │
│   ← 選択されたタブの内容                            │
│                                                     │
└─────────────────────────────────────────────────────┘
```

- 上部に2タブを横並び、クリックで切替
- URL に `?view=calendar` / `?view=heatmap` で状態保持
- ブラウザの戻る/進むで履歴を残す

→ **マニ推奨**。最もシンプル。状態が明示的。

#### 案T2: 上下並置（同時表示）

```
┌─────────────────────────────────────────────────────┐
│  カレンダー本                          [折りたたみ] │
│  (案B 月カレンダー)                                 │
├─────────────────────────────────────────────────────┤
│  長尺ヒートマップ                      [折りたたみ] │
│  (案C 日次連続)                                     │
└─────────────────────────────────────────────────────┘
```

- 両方を一画面で。折りたたみで縮小可能
- 「月単位の俯瞰」と「連続地形」を同時に見たい場合に便利

→ Mac/iPad で**画面が十分広い前提**で有用。デフォルトでは案T1 推奨。

#### 案T3: 横並び（split view）

```
┌──────────────────┬──────────────────────┐
│  カレンダー本     │  長尺ヒートマップ    │
│  (月)            │  (連続)              │
│                  │                      │
└──────────────────┴──────────────────────┘
```

- 左右分割。クリックで連動（カレンダーで選んだ日が右でハイライト）
- 情報密度は最高だが**狭い画面で破綻**

→ 後工程の **「連動ハイライト」機能**を入れるとき再検討。

### 連携の仕掛け（タブ切替時の同期）

- 本タブで「3/5(Thu)」を選択している状態でタブ切替 → 別タブでも 3/5 にスクロール・ハイライト
- 別タブで日次セルをクリック → 本タブ側で該当月へジャンプ＋ハイライト

→ 後工程の話。v0.2 では**UI構造のみ確定**、連携実装は後回しでよい。

## 6. エントリーロジック混入セルフチェック v0.2 更新

### v0.1 からの変更点

| 要素 | v0.1 判定 | v0.2 判定 | 変化 |
|---|---|---|---|
| 「凪/弱/中/HOT/強」バッジ | ⚠️ 要注意 | ❌ **完全除去** | リスク消滅 |
| ADX35 警告色 | ⚠️ やらない | ❌ **やらない（継続）** | 維持 |
| パターン別色分け | ⚠️ やらない | ❌ **やらない（継続）** | 維持 |
| 背景濃淡 | ✅ 環境観測のみ | ✅ 環境観測のみ | 維持 |
| 色相 DI+/- | ✅ 環境観測のみ | ✅ 環境観測のみ | 維持 |

### v0.2 新規リスク評価

#### リスクR1: 背景濃淡の「閾値感」がUIに滲む

- **症状**: 「濃い背景＝高ADX＝危ない」という解釈が読み手に生まれる可能性
- **対策**: ✅ **連続グラデーション**を徹底。段階色（5段階等）にしない。
- **判定**: 設計で対処可。リスク低。

#### リスクR2: シグナルマーカーの形状が「推奨度」に見える

- **症状**: ▲ より • が「弱い推奨」と読まれる可能性
- **対策**: ✅ 凡例で「· = 不発、• = 候補、▲ = 発生」と**事実記述だけに限定**。「推奨」「強度」のような言葉を凡例から排除。
- **判定**: 凡例設計で対処可。リスク低。

#### リスクR3: 別タブ長尺ヒートマップで「地形がそのまま戦略になる」誤読

- **症状**: 長尺で「ADX が高い帯でシグナル集中」を見ると、「ADX 高 = エントリー OK」と読みかねない
- **対策**: ✅ **環境とシグナルと結果を別の行に物理分離**（v0.1 から継続）。さらに**シグナル行と結果行の上に強い仕切線**を入れ、「環境はあくまで背景」と視覚で示す。
- **判定**: 仕切線設計で対処。継続監視。

#### リスクR4: 「★評価」が個別判断の優劣評価に見える

- **症状**: ★3 = 良い、★1 = 悪い、という個別1トレード評価に読まれる
- **対策**: ✅ ★の意味を**「結果の規模（MAE/MFE スケール）」**に固定。**「判断の良し悪し」ではない**と凡例で明示。
- **判定**: 凡例で対処。要監視（あろさんの感覚と合うか後で確認）。

#### リスクR5: 環境レーン70%拡大で「環境を主役に」が「環境を見て判断」に滑る

- **症状**: 視覚優先度が高すぎて「環境が良い日にエントリーすればいい」という近道思考を誘発
- **対策**: ✅ **シグナル行を独立に持つことで「環境が良くてもシグナルが出なければトレードはない」**を構造的に維持。シグナル行の存在自体が「環境 ≠ 判断」を保証。
- **判定**: 構造で対処済。リスク低。

### v0.2 セルフチェック総括

> HOT系ラベルの完全除去により、**v0.1 で最大の混入リスクだった「カテゴリ閾値の UI 出力」が消滅**。
> 残るリスクはすべて**凡例設計と仕切線**で対処可能。
> 環境レーン70%拡大は、シグナル行の物理的独立により混入リスクは増えていない。

→ **マニ判定: 混入リスクは v0.1 より低下**。おぱの番人レビュー OK ならこの構造で進めて問題なし。
