## 0. 研究目的（絶対固定）

このパックは日次研究カレンダー（マニ実装）のデータソース拡張。

研究目的:
- 「どの市場環境で期待値が発生しているか」
- 「自分が正しい局面でどれくらいトレードできていたか」
- 「正しい環境でもどれくらいエントリーミスがあって負けてるか」

### 禁止事項
- 勝率分析 / PF分析 / 月別集計 / 損益集計を**目的化しない**
- データ取得そのものを目的化しない
- パターン別重み付け、加熱帯ペナルティ等の**判断ロジック混入禁止**

### 提案ルール
- 取得項目・粒度の拡張を提案する場合、「何の仮説検証に使うか」「そのデータで何が判定できるか」を明示
- 説明できない取得項目は実装しない

## 2. C1: 非トレード日 × 48h MFE/MAE 取得スクリプト

### 目的
- 「もし入ってたら 48h MFE/MAE はどうだったか」を**全営業日に拡張**
- 既存 `trades_enriched_full.csv` のトレード日30件を全営業日に拡張

### 仕様
- ファイル名: `XAUUSD_Daily_MFE_MAE_v1.mq5`
- 入力: なし（MT5から直接取得）
- 出力: `mt5_data/daily_mfe_mae_48h.csv` (UTF-8-sig)
- 配置: `MQL5/Scripts/Examples/`

### 仮想エントリー設定
- 全営業日 **14:00 JST 仮想エントリー**（東京時間メイン、あろさんスタイル準拠）
- **BUY/SELL 両方記録**（マニ側で評価）
- サーバーGMT offset 自動算出は既存 Trade_Snapshot_Builder.mq5 v1.1 のロジック流用

### 出力カラム
```
date                  # YYYY-MM-DD (JST)
virtual_entry_jst     # YYYY-MM-DD HH:MM:SS (JST 14:00)
virtual_entry_price   # その時刻のH1 close
buy_mfe_usd           # BUY 仮想 48h MFE (USD)
buy_mae_usd           # BUY 仮想 48h MAE (USD)
sell_mfe_usd          # SELL 仮想 48h MFE (USD)
sell_mae_usd          # SELL 仮想 48h MAE (USD)
bars_traced           # 追跡したH1足数（参考）
```

### アルゴリズム
1. 起動時パラメータ: 取得期間（デフォ: 直近120日、可変）
2. 各営業日について:
   - 14:00 JST のH1足を特定（土日・祝日スキップ）
   - その close を仮想エントリー価格
   - 48時間後（H1足×48本）の終了時刻まで H1足を順次トレース
   - 期間中の Max High - entry = MFE (BUY側)
   - entry - Min Low = MAE (BUY側)
   - SELL 側は符号逆転（entry - Min Low = MFE, Max High - entry = MAE）
3. USD換算（XAUUSD は 1pip = 1 USD/oz 概算でOK、マニ確認時に微調整可）

### 注意
- **「日次変動の捏造禁止」原則を守る**: 事実情報の拡張のみ、推定値・補完値は禁止
- 既存 `signals/Trade_Snapshot_Builder.mq5` v1.1 の三層MAE/MFE ロジック（特に48時間追跡部分）を流用可
- BUY/SELL 両方記録するのは「マニ側で評価する自由度」を持たせるため、コーは判断しない

## 3. C2: 日次集計 D1/H4/H1 ADX/DI/ATR

### 目的
- 現状 `weekly_waves.json` は週次集計サマリ → カレンダーで同週内 5日同値が並ぶ
- **日次粒度で景色が変化**する必要あり
- D1 ADX22, H4 ADX46, H1 ADX32 等を日次集計

### 仕様
- ファイル名: `XAUUSD_Daily_Aggregate_v1.mq5`
- 出力: `mt5_data/daily_aggregate.csv` (UTF-8-sig)
- 配置: `MQL5/Scripts/Examples/`

### 出力カラム
```
date                      # YYYY-MM-DD (JST)

# D1 (1値/日、日足終値時点)
d1_adx22, d1_di_plus, d1_di_minus, d1_di_spread, d1_di_dir
d1_atr22, d1_atr42, d1_atr22_42_ratio

# H4 (1日6本→3軸: max/close/mean)
h4_adx46_max, h4_adx46_close, h4_adx46_mean
h4_di_plus_close, h4_di_minus_close, h4_di_spread_close, h4_di_dir
h4_atr8_close, h4_atr46_close, h4_atr8_46_ratio_close

# H1 (1日24本→3軸: max/close/mean)
h1_adx32_max, h1_adx32_close, h1_adx32_mean
h1_di_plus_close, h1_di_minus_close, h1_di_spread_close, h1_di_dir
h1_atr16_close, h1_atr32_close, h1_atr16_32_ratio_close
```

### マニからの利用方法
- **max** = 「その日伸びた瞬間」評価（あろさん流の伸び）
- **close** = 安定構造評価
- **mean** = 平滑評価
- マニ側で使う指標は自由に選択

### 留意
- 「最大」「終値」「平均」の3軸全部出して、マニが選べる状態にする
- パターン別重み付け、加熱帯ペナルティ等の判断ロジック混入禁止
- ADX/DI/ATR 周期は CLAUDE.md の「XAUUSD 指標周期設定（確定値）」に従う:
  - D1: ADX22, ATR22, ATR42
  - H4: ADX46, ATR8, ATR46
  - H1: ADX32, ATR16, ATR32

## 4. C3: v4 mq5 シグナル発火ログ CSV出力

### 目的
- 現状: トレード発生日 = シグナル発火日と仮定
- 修正必要: 「シグナル発火したがエントリーしなかった日」「シグナル発火しなかったがエントリーした日」を判別
- → **取り逃がし / 過剰反応 の検出**（あろさん研究目的の核）

### 仕様
- 既存 `signals/ATR_WidthSignal_v4.mq5` に**CSV書き込みロジック追加**
- 出力: `mt5_data/v4_signal_log.csv` (UTF-8-sig)
- 発火条件発生時に CSV に1行追記（append モード）

### 出力カラム
```
fire_time_jst         # YYYY-MM-DD HH:MM:SS (JST)
direction             # BUY / SELL
pattern               # PatA / PatB / PatC / PatD / Sweet
h1_adx32, h4_adx46, h4_pct_above20
filter_passed_flags   # F1-F9 通過状態 (例: "F1F2F3F5F7F8" or bitmask)
entry_price           # シグナル発火時のH1 close
note                  # 任意（拡張用）
```

### 注意
- v4 mq5 のフィルター F1-F9 は**既存実装通り維持**（新規フィルター追加じゃない、ログ追加のみ）
- インジケータとしてリアルタイム動作中なので:
  - リアルタイム書き込み（OnCalculate内、新規発火検出時）
  - または H1足確定時の書き込み
  - どちらでも可、コー判断
