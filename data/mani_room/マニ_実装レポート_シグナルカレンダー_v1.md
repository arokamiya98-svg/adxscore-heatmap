## 2. 設計判断（指示書外でマニが決めた点）

1. **カレンダーは月〜土の6列**（daily_calendar は月〜金5列）
   - signal_fires.csv は JST 基準で、サーバー金曜深夜 = JST 土曜の発火が **31件** 存在
   - 5列流用だと土曜発火が欠落して「389件全件」の完了条件を満たせない
   - 日曜発火は 0件 → 日曜列は置かない
2. **ドットは全件描画（+n 集約なし）**
   - 最大14件/日（2025-03-14）。flex-wrap の小グリフで収まる
   - 集約するとフィルター切替時の件数管理が複雑化し「欠落ゼロ」の検証可能性が下がる
3. **ドット描画は in-month セルのみ**
   - 週端の隣月セル（.outside）にドットを置くと月マタギで二重描画され389カウントが壊れるため
4. **ドリルダウンは右サイド固定ドロワー**（16ヶ月の縦長カレンダーをスクロールしながら日クリック参照）
   - 表示フィルターと連動（フィルターで隠れた発火は「n件非表示中」とタイトルに明示、勝手に消さない）
5. **MFE/MAE ステップバーの正規化はカード内最大値基準**（「伸び方の形」を見る用。全期間比較ではない旨を drill-note に記載）

## 3. 指示書準拠の確認

- 色 = v4 矢印色テーブル完全一致（10色全て、§5検証[v4色]参照）。**色は方向に使わない**
- 方向 = 形（▲BUY / ▼SELL）
- pass_all=FALSE = `opacity: 0.3`（.fire-dot.suppressed）+ ドリルカードも破線・減光
- 環境値は段階ラベル優先（atr_zone / h1_adx_zone / h4_adx_zone / cross_dir）、生値は括弧内の補助
- 実トレードオーバーレイ: デフォルトOFF、ON時は枠線+方向マーク（▲/▼）のみ。損益・評価は一切出さない
- 月ヘッダは発火件数（pass/抑制）のみ。**損益/勝率サマリーなし**
- CSV読込は全て `encoding="utf-8-sig"`。さらに先頭キーが `fire_id` であることを assert（BOM混入の即時検出）

## 5. HTML構造の要点（レビュー用）

### 主要 id / class

| セレクタ | 役割 |
|---|---|
| `.filter-bar`（sticky） | 上部フィルター。パターンcb / pass切替 / 方向切替 / 実トレードトグル / 表示件数 |
| `.pat-cb[data-pat]` | PatA〜E チェックボックス（OFF時 .off で減光） |
| `.seg-btn[data-pass]` | 「全発火 / pass_all のみ」2状態切替 |
| `.seg-btn[data-dir]` | ALL / BUY / SELL 切替 |
| `#trade-toggle` | 実トレードオーバーレイ ON/OFF（body.show-trades で発動） |
| `#visible-count` | 表示中件数（pass/抑制の内訳付き、フィルター連動） |
| `.month` > `.week-cells` | 月ブロック > 週行（6列グリッド、月〜土） |
| `.cell[data-date]` | 日セル（in-monthのみ。.has-fires でクリック可、.has-trade で枠線対象） |
| `.fire-dot` | 発火グリフ。`data-fid / data-pat / data-dir / data-pass` 属性。`.suppressed`=opacity0.3、`.fhide`=フィルター非表示 |
| `.trade-mark` | 実トレード方向マーク（show-trades 時のみ表示） |
| `#drill` / `#drill-title` / `#drill-body` | 右サイド固定ドロワー（発火カード列挙） |
| `.fire-card` | ドリルカード。`.suppressed`=破線+減光。`.mm-steps`=MFE/MAE 4段ステップバー |

### JS関数構成（全て `<script>` 1本、インラインJSON）

| 関数/定数 | 役割 |
|---|---|
| `FIRES` / `PAT_COLORS` | インライン埋め込みデータ（389件 / v4色テーブル） |
| `byDate` | date → fires のインデックス |
| `state` | フィルター状態（pats: Set / pass: "all"\|"pass" / dir: ALL\|BUY\|SELL） |
| `fireVisible(pat, dir, pass)` | フィルター判定の単一ソース（ドットとドリル両方が使う） |
| `applyFilters()` | 全 .fire-dot に .fhide 適用 + #visible-count 更新 + 開いてるドリル再描画 |
| `mmSteps(f)` | MFE/MAE 12/24/36/48h ステップバー生成（カード内max正規化、bars_traced<48 は注記） |
| `fireCard(f)` | 発火カードHTML（時刻JST+server併記 / 環境段階ラベル / F1〜F9ヒット明示） |
| `renderDrill(dateStr)` | ドロワー描画 + セルハイライト（.drill-open） |
