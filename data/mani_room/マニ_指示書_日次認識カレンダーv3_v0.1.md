## 4. タブ1: 全体像カレンダーの仕様

### 4.1 セルの視覚要素（これだけ。優先順）

1. **背景色** = 環境の強度と方向（§4.2）
2. **MFE/MAE バー** = 値動きの形（v2.0 のバー表示踏襲）
3. **シグナルドット** = v4発火（§4.4）
4. **トレードマーク** = 実弾（§4.5）

**削除するもの**: H1 ADX 数値テキスト、金額表記、その他のセル内文字情報。
ホバー（title）に日次サマリーは残してよい（スコア値・発火数・トレード有無など）。

### 4.2 背景色ロジック（v3 の核心変更）

```
強度（明度・彩度の段階） = 日次合成スコア（calc_daily_score 現行式そのまま、score_step 段階化続行）
方向（色相 青/赤）       = H1 DI spread 日次平均の符号   ← h4_di_spread から変更
確信度（彩度補正）       = H1 と H4 の DI 方向が一致 → フル彩度
                          不一致 → 彩度を一段落とす（迷い相場の表現）
```

- **凪/レンジ判定: スコアのみ**（`h4_adx46 < 15` 条件は撤去）
- 低スコアのグレー処理（2段階の濃淡）は**現行の見た目を踏襲**（あろさん指定）
- **ラベル表記を「凪」→「レンジ」に変更**。理由: この判定は ADX/スコアベース = RANGE。凪は ATR 収束の用語で別概念（メモリ: nagi-vs-range-distinction）。グレーの色味は変えない、文字だけ
- 連続グラデーション禁止（段階化維持）

### 4.3 期待される改善（レポートで実例確認すること）

直近の急落局面（前週買い相場→急落）で、v2.0 は色の赤転が2日遅れた（h4_di_spread のラグ）。v3 は H1 DI ベースなので**急落の翌日には赤系に変わる**はず。レポートに同期間の v2.0 と v3 の色変化比較を記載。
