## 3. U1: 円グラフ4つクリッカブル化

### 3.1 対象（4種類）

`generate_daily_calendar.py` L2073-2078 の `pies_def`:

| グラフ | カテゴリ | クリック後のフィルタ |
|---|---|---|
| 反省タグ別 | PAT-A/B/C/D/ATR収束底/その他/未分類 | `decision_tag == 選択値` |
| 勝敗別 | 勝ち/負け/建値 | `result == 選択値` |
| D1フェーズ別 | BU/PD/RANGE/その他 | `d1_phase == 選択値` |
| 曜日別 | 月〜金 | `weekday == 選択値` |

### 3.2 設計フロー（あろさん原文）

```
グラフ
  ↓ 気になる扇形をタップ
下の詳細一覧（drilldown-table）が切り替わる
  ↓ 列ヘッダタップ
ATR 絶対値で並ぶ
  ↓ 上から眺める
「ATR 何帯から MFE が変化するか」が見える
```

### 3.3 扇形クリック実装方針

- 既存 `render_pie()` (L2017) の `<path>` 要素に `data-pie-key="..."` と `data-pie-value="..."` を追加
- `class="pie-slice-clickable"` 等で hover/active CSS（薄く光るとか、マニ判断）
- JS で `addEventListener("click", ...)` → `drilldown-wrap` に該当データを流す

### 3.4 「現在のフィルタ」表示

`drilldown-title` を活用:
```
反省タグ別: PAT-C (N=4)
勝敗別: 勝ち (N=12)
D1フェーズ別: BU (N=18)
曜日別: 月 (N=6)
```

### 3.5 フィルタ解除

- 同じ扇形を再クリック → 解除
- または「全件表示」ボタンを drilldown-head に追加
- マニ判断 OK

### 3.6 複数フィルタの扱い

**シンプル方針**: 1グラフ1フィルタ、**最後にクリックされたものだけ有効**。
- 円グラフAでPAT-C選択中に円グラフBで「勝ち」クリック → 「勝ち」のみ表示（PAT-C フィルタは解除）
- 複合AND（PAT-C × 勝ち）は v0.2 では未実装。将来要望が出たら追加。
