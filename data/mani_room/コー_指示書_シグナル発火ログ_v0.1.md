## 0. 目的（固定・変更禁止）

**稼働中の v4 シグナルの発火を過去再現して記録し、あろさんがシグナルへの理解を深めるための可視化データを作る。**

- v4 本体は**一切変更しない**（読むだけ）
- シグナルの改良・最適化は目的に含まない（8〜9月メンテの管轄）
- 研究ルール準拠: シグナル評価の文脈での勝敗・MFE/MAE は OK。結果フィッティング禁止

## 1. 成果物

`Signal_Fire_Logger_v1.mq5`（Script 型、`signals/` にミラー）
出力: `MQL5/Files/signal_fires.csv`

## 2. ロジック源泉の絶対原則

```
発火ロジック・フィルター条件は signals/ATR_WidthSignal_v4.mq5 から忠実移植。
BT世代2 (ATR_WidthSignal_BT_v3bywavelog_gen2.mq5) のロジックは参照しない。
（Script 骨格・CSV出力・MFE/MAE走査の書き方は既存 daily 系 mq5 を参考にして良い）
```

v4 と同じ判定になることが命。周期・閾値は v4 の input デフォルト値を**定数として**持つ（input で変えられる必要なし。変えられると「稼働中の v4 の再現」でなくなる）。

## 3. 仕様

### 3.1 走査

- 銘柄/TF: XAUUSD H1（v4 の稼働チャートと同一）
- 期間: **2025-03-01 00:00 〜 実行時点**（input で変更可にして良い）
- 各 H1 確定バーで v4 の 5パターン × BUY/SELL 判定を実行
- multi-fire 仕様も v4 と同一（同一バー複数パターン発火 = 複数行）

### 3.2 フィルターは「ラベルのみ・除外しない」

発火は**フィルター前の全件**を記録。F1〜F9 は各行に TRUE/FALSE 列で付与:

| 列 | v4 input 対応 |
|----|--------------|
| f1_none_sell | Filter_F1_NoneSell |
| f2_patb_midh_sell | Filter_F2_PatBMidHSell |
| f3_patd_pd_buy | Filter_F3_PatDPdBuy |
| f4_patc_up_none_midh | Filter_F4_PatC_UpNoneMidH |
| f5_patb_up_bu_midh | Filter_F5_PatB_UpBuMidH |
| f6_patc_up_pd_midh | Filter_F6_PatC_UpPdMidH |
| f7_tight_sell | Filter_F7_TightSell |
| f8_patc_none_sell | Filter_F8_PatC_NoneSell |
| f9_pata_weakup_sell | Filter_F9_PatA_WeakUpSell |

- TRUE = そのフィルターが抑制対象と判定（= v4 実機なら非表示になる発火）
- `pass_all` 列: 9本すべて非抑制なら TRUE（= v4 実機で矢印が出た発火）

### 3.3 エントリー基準と MFE/MAE（C5 原則）

- `entry_price` = **発火バーの close**。発火はバー確定時点で起きるので、これは「その瞬間に利用可能な値」であり C5 問題なし
- 指標値も v4 と同じ確定バー基準を厳守（v4 の shift 規約から一切ずらさない）
- MFE/MAE: 発火バーの**次の H1 バーから 48本**走査、12/24/36/48h の4セグメント（C4 の `TraceMaeMfe_Segmented` と同じ方式）
- 発火方向基準で記録（BUY発火なら BUY視点の MFE/MAE。反対方向は対称なので列不要）

### 3.4 CSV 列構成（案・コー裁量で微調整可）

```
[キー]      fire_id, date, time_jst, time_server, pattern(A-E), direction(BUY/SELL)
[価格]      entry_price
[環境]      v4 が判定に使う状態量をそのまま:
            h1_atr16, h1_atr32, h1_atr_ratio, atr_zone(LOW/NORMAL/HIGH),
            h1_adx32, h1_di_plus, h1_di_minus,
            h4_atr8, h4_atr46, h4_adx46, h4_di_plus, h4_di_minus,
            h4_phase(BU/PD等), cross_dir(UP/DOWN/NONE),
            pair_state(EXPAND/FLAT/CONTRACT), d1_atr22, d1_atr42, d1_adx22
[フィルター] f1〜f9 (TRUE/FALSE), pass_all
[追跡]      mfe_12h, mae_12h, mfe_24h, mae_24h, mfe_36h, mae_36h,
            mfe_48h, mae_48h, mfe_bar_idx_48h, mae_bar_idx_48h, bars_traced
```

### 3.5 出力規約

- `FILE_WRITE|FILE_BIN` + BOM 手動書込（UTF-8-sig、daily 系と同じ）
- 全量上書き再生成（追記しない）
- JST 変換・DST は Trade_Snapshot_Builder と同じ規約

## 4. 運用（参考情報）

- 実行後は auto_sync_daily.sh が検知して可視化まで自動（おぱ側で監視対象に追加する）
