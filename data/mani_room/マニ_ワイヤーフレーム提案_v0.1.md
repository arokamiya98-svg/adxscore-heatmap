## 0. 設計の出発点（マニの理解）

### 現状の問題（再確認）

`scripts/generate_trades_calendar.py` を読み込んで原因特定:

```python
week_map = {w["week"]: w for w in weeks}     # ← 週単位辞書
def get_rec(d): return week_map.get(date_to_iso_week(d))  # ← 同じ週の全日に同じ値
```

→ 月曜〜金曜まで全部「ADX39 ⇄+4」と並ぶ。**日次の変化がゼロ**。
→ これは「週次マクロの移植」であって「日次研究」ではない。

### この提案で解こうとしていること

1. **環境強度を主役にする**（背景の濃淡で「強い偏り」が一目で伝わる）
2. **市場環境・シグナル品質・結果を視覚的に分離する**（3レイヤーが混ざらない）
3. **日次の変動が見える構造**（同じ値が並ばない）
4. **売買判断ツールではない**ことを構造で守る（エントリーロジック混入を排除）

## 1. 3レイヤー分離の共通設計思想（4案すべての前提）

```
レイヤー1: 市場環境（主役・最も大きく）
  ├ H4/H1 ADX強度（環境強度の主因）
  ├ DI バランス（偏りの方向と強さ）
  └ 構造ラベル BU/PD/凪/凪離脱/収束底（小さなバッジ）

レイヤー2: シグナル品質（補助・小さく）
  ├ v4 シグナル発生有無
  └ パターン種別（必要に応じ）

レイヤー3: 結果（補助・最も控えめ）
  ├ MAE/MFE（固定48h）
  └ 損益・★評価
```

### 視覚優先順位ルール（4案共通）

| 優先順位 | 要素 | 表現 |
|---|---|---|
| 1 | 環境強度（H4 ADX×DI） | **背景**（濃淡・色相） |
| 2 | 構造ラベル | 小バッジ（コーナー） |
| 3 | シグナル発生 | 点・線・小マーカー |
| 4 | 結果 | フッターまたはホバー時詳細 |

### 「日次の独立性」確保ルール

すべての案で、**セルの主役表現は日次データに直接バインド**する。
週単位の値を全日に塗り回すパターンは禁止（現状の失敗を繰り返さない）。


### セル詳細（拡大図）

```
┌────────────┐
│ 5  ★BU     │  ← 日付 + 構造ラベル(小バッジ右上)
│ ██████████ │  ← 環境強度バー(背景濃さ): ADX値が高いほど濃く
│ ██████████ │     色相: DI+優勢=青系 / DI-優勢=赤系 / 拮抗=灰
│ ─────────  │
│  ▲ +250    │  ← シグナル▲ / MAE-MFE結果
│  L0.05     │
└────────────┘
```

### 主役の表現方法
- **背景濃淡**: ADX値を 0〜100% にマップし、不透明度として表現
- **色相**: DI+ - DI- の符号で青/赤を切り替え
- **上半分の高さ比率**: セル全体の 60% を環境強度に割り当て（視覚的主役）

### 補助情報の置き場所
- 構造ラベル: 右上の小バッジ（既存資産から継承）
- シグナル: 下半分左の小マーカー
- 結果: 下半分中央〜右の小数字

### Pros
- 既存 `trades_calendar.html` のレイアウトと整合性が高く、移行コストが低い
- セルを見た瞬間「濃い＝強い環境」がわかる
- 背景の色相変化で DI 方向もすぐ読める

### Cons
- セルサイズが小さいため、3レイヤーの分離が「上下分割」程度に留まる
- 環境強度を背景に頼るため、複数指標（H4・H1分離）の表現が難しい
- 結果情報が小さくなりがち（情報密度の限界）

### 「今日は偏りが強い日だった」の一目視認性
**◯ 良い**: セル全体の濃さが一目でわかる。1ヶ月見渡したときに濃い日・薄い日が地形のように見える。

- 長尺スクロールで「数週間〜数ヶ月の環境推移」を地形として見渡せる

- 月単位の「カレンダー」感が失われる（土日が空白セルになる）

- セル幅が狭いため、結果の詳細表示が苦しい（クリックで詳細パネル展開などの仕掛けが必要）

**◎ 最高**: H4 ADX 行と H1 ADX 行を横に見るだけで、その日が他の日と比べてどれくらい強かったかが地形として瞬時にわかる。**「今日の偏り」が「文脈の中での今日」として読める**のが最大の強み。

### 主役の表現方法
- **環境バッジ**（[凪]/[弱]/[中]/[HOT]/[強] + ↑/↓/⇄）を大きく中央上部に配置
- 補助的に ADX 実数値 + DI スプレッドの数値を併記
- **H1 ADX スパークライン**（その日のH1足ごとのADX推移を8〜24本の小ヒストグラム）
  → 「日次の中でも変動があった日／一日中強かった日」が読み分けられる

### 補助情報の置き場所
- 中段: D1/H4 構造ラベル
- 下段: シグナル → 結果（縦に積む）

### Pros
- **環境バッジ + スパークラインで「環境強度の質感」**まで伝わる（強度だけでなく安定性も）
- バッジ表記により、ぱっと見の判別性が極めて高い
- 数値も併記されるので、計算式検証時の確認がしやすい

### Cons
- スパークライン用に H1足 ADX の時系列データが必要（コー案件の MT5 集計仕様に追加要素）
- セル1個あたりの情報量が多く、情報密度が上がりすぎる懸念
- カテゴリ分類（凪/弱/中/HOT/強）の閾値設計が UI に直接出るため、計算式変更時の影響範囲が広い
- **「凪/弱/中/HOT/強」のバッジ自体がエントリーロジック寄りに見える可能性**（→ セルフチェック⚠️ 後述）

### 「今日は偏りが強い日だった」の一目視認性
**◯ 良い**: バッジで瞬時にカテゴリがわかる。ただし「カテゴリ化された見え方」になるので、案Cのような「文脈の中での今日」感は薄い。

## 7. マニ推奨案と理由

### 推奨: **案B（3レーン水平分割）をベースに、案C（環境地形）の思想を一部取り込む**

#### 理由

1. **指示書の核心要求**「市場環境・シグナル品質・結果を分離して観測」を視覚構造そのもので実現できるのが案B
2. レーンが明示的に区切られているため、「環境だけを目で追う」「シグナルだけを目で追う」が物理的に可能
3. 案Cは思想として最も美しいが、月カレンダー形式から離れすぎてあろさんの感覚との接続が薄くなる。**まずは案Bで「日次カレンダー」を確立し、別ビュー（案C型の長尺ヒートマップ）を後工程で追加**するのが現実的
4. 案Aは移行コストが最も低いが、3レイヤー分離が弱く「焼き直し脱却」感が薄い
5. 案Dはバッジ自体に環境分類のロジックが内包されてしまうため、計算式詳細を後工程にする方針と相性が悪い

#### 推奨案のレイヤー比率（最終提案）

```
セル全高を 100% として:
  日付ヘッダ           :  10%
  レーン1 (市場環境)    :  55%   ← 主役。背景濃淡 + ADX値 + DI + 偏り強度ラベル
  レーン2 (シグナル品質):  15%
  レーン3 (結果)        :  20%
```

#### あろさんへのオプション提示

- **オプション1**: 推奨どおり案Bで開始
- **オプション2**: 案A（天気予報グリッド）で軽量に始めて、後で案Bに拡張
- **オプション3**: 案Cの長尺ビューを「日次研究カレンダー」の **別タブ**として併設（heatmap_v14 の日次版）
