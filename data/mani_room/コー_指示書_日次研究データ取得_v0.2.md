## 5. C4: 時間別 MFE/MAE @12/24/36/48h 推移取得 ★★★ 新規

### 5.1 目的

#### 研究上の必然性

現状（C1）の MFE/MAE は **48h時点の最大値のみ**。これでは:
- 「ATR 高い時、12h 以内に勝負がつく」のか「48h 粘ると伸びる」のか **判別不能**
- 「ATR 低い時、遅く伸びる」のか「最初から伸びない」のか **判別不能**
- → ATR帯 × 時間推移 × 勝敗 のクロス探索が**不可能**

C4 はこれを解決するため、**12h / 24h / 36h / 48h 各時点の累積最大MFE・最大MAE**を取得する。

#### マニ側での利用

```
[詳細テーブル ATR 昇順ソート]
 H1 ATR 7.1 → 12h MFE / 12h MAE / 24h MFE / 24h MAE / ... / 48h MFE / 48h MAE
 H1 ATR 7.8 → 同上
 H1 ATR 8.3 → 同上
 ...
 H1 ATR 18.4 → 同上

→ ATR 上昇に従って「MFE/MAE のピーク時刻」が早まる/遅れる構造が見える
→ 「ATR帯ごとの典型的な伸び方」が時系列パターンとして抽出される
```

→ これが「環境（ATR帯）× 結果の時間構造」の研究目的の核。

### 5.2 C4-A: Trade_Snapshot_Builder.mq5 改修

#### ファイル
- `signals/Trade_Snapshot_Builder.mq5` v1.1 → **v1.2 化**

#### 改修内容

既存の 48h 三層 (H1/H4/D1) MFE/MAE 追跡ロジックを **4時点記録に拡張**。

#### 新規出力カラム（追加）

```
# H1 三層 (1h足ベース、12/24/36/48 時間 = 12/24/36/48 bar)
h1_mfe_usd_12h, h1_mae_usd_12h
h1_mfe_usd_24h, h1_mae_usd_24h
h1_mfe_usd_36h, h1_mae_usd_36h
h1_mfe_usd_48h, h1_mae_usd_48h   ← 既存（リネーム検討: 48h明示）

# H4 三層 (4h足ベース、3/6/9/12 bar = 12/24/36/48 時間)
h4_mfe_usd_12h, h4_mae_usd_12h
h4_mfe_usd_24h, h4_mae_usd_24h
h4_mfe_usd_36h, h4_mae_usd_36h
h4_mfe_usd_48h, h4_mae_usd_48h   ← 既存（リネーム検討）

# D1 三層 (1日足ベース、12h以下は「未達」扱い、24/48h のみ意味あり)
d1_mfe_usd_24h, d1_mae_usd_24h
d1_mfe_usd_48h, d1_mae_usd_48h   ← 既存（リネーム検討）
# 注: D1 の 12h/36h は同bar内サンプル数が少ないため取得しない
```

#### 計算ロジック

各時点 N（N=12,24,36,48）について:
- エントリー時刻から N 時間以内の H1足/H4足/D1足 を順次トレース
- 期間中の **累積最大** High から entry を引いた値 = MFE（BUY 側）
- entry から **累積最小** Low を引いた値 = MAE（BUY 側）
- SELL 側は符号逆転

**累積値の意味**: 「N時間以内のどこかで最大に到達したか」を見るための値。N時間時点の瞬間値ではない。

#### bar_idx 列の継続

既存の `h1_mfe_bar_idx_48h` / `h1_mae_bar_idx_48h` 等はそのまま（最大に到達した時刻参考）。
