## 1. 認識ターゲット宣言（あろさんの言葉・原文ベース）

> **「純粋に認識ツールの土台で、どう相場が動いて、そこにシグナルとトレード実弾がどう機能したか。認識したいのはこれに絞っていきたい。」**

このカレンダーが答える問いは3つだけ:

1. **相場はどう動いたか**（環境認識 = 色と MFE/MAE バー）
2. **シグナルはどう機能したか**（v4発火の有無・種類・通過）
3. **トレード実弾はどう機能したか**（執行の有無・方向・結果）

この3つに寄与しない視覚情報は削るか格下げする。

## 2. ツール3層の位置づけ（認知システム全体図）

```
週次ヒートマップ (heatmap_v14)     = マクロ認識（週のリズム）
        ↓ 週次で見たものを日足に落とす
日次認識カレンダー (trades_calendar) = 環境認識の主戦場 ★今回のv3対象
        ↓ 日をクリック/ホバーで掘る
統合評価シート (signals_calendar)    = シグナル×実行の詳細認知
```

あろさん評価: 「日次の情報としてはかなり良い。ADXがあった = トレードに良い環境だったことと方向が一眼で色で分かる」← この強みは維持する。

## 3. 現状の問題（コード根拠つき）

### 3.1 色の反応が相場より2日遅れる

**観察事例（あろさん）**: 13〜14日に大きく下げたのに、カレンダーの色は2日くらい後に赤転。前週の買い場のノイズが消えるまで色が変化していない。

**コード上の原因** (`generate_daily_calendar.py` の `h4_bg_style`):

| 要素 | 現在のソース | 問題 |
|---|---|---|
| 色相（青/赤） | `h4_di_spread`（H4 DI46） | ADX/DI 46周期は平滑化が強く、方向転換に数日遅れる。**ラグの主犯** |
| 凪判定 | スコア < 15 **または** `h4_adx46 < 15` | H4側の条件が同様のラグを持ち込む |
| 明度・彩度段階 | 日次合成スコア（H1 50% + H4 30% + D1整合 20%） | **これは速い。問題なし** |

**結論**: スコアの方が早いリズムで相場を反映できている。色付けの基準をスコア側のリズムに揃える。

## 4. 変更要件

### R1: 色付けのスコアベース化（最重要）

- 強度段階（明度・彩度・グロウ）: 日次合成スコアのみで決定（現行の段階化テーブル続行、連続グラデ禁止は維持）
- 凪判定: **スコアのみ**（`h4_adx46 < 15` の条件を撤去）
- 色相（青/赤の方向）: H4 DI から、より速い方向ソースへ変更 → **§6-Q1 で確定**
- 期待効果: 急落の翌日には色が追随する（H4ノイズの残像が消える）

### R2: セル内テキストの削減

- **H1 ADX 数値表記を削除**
- セルの視覚要素は「**色（=環境の強度と方向）+ MFE/MAE バー（=値動きの形）**」の2つだけにする
- あろさんの言葉: 「純粋に色とMAE/MFEバーだけで相場の状況を知れる」

### R3: 金額表記の視覚格下げ

- 現状: トレード日が「青=買いポジション」+ 金額表示
- 変更: **金額は視覚情報から格下げ**（削除ではなく、ホバー/ドリルダウンの中へ降格。一覧面には出さない）
- 研究ルールとの整合: 元々集計禁止。個別表示も「あってもいいけど格下げ」があろさんの指定

### R4: トレード表示を評価シート流に

- トレード日マークは軽量化（評価シート v2 と同じ言語: 枠線+方向▲▼）
- **カーソルを当てたら（ホバー/クリック）詳細が出る**: 評価シート v2 のトレードカード（時刻 / 方向 / pips / スタイル / ★ / タグ / MFE/MAE 12→24→36→48h 推移 / 新規理由折りたたみ）を流用
- 実装は signals_calendar v2 のドロワー/カード構造をコード流用してよい

### R5: シグナル発火表示の追加

- 日次カレンダーにも v4 発火を表示（データ: `mt5_data/signal_fires.csv`）
- 視覚言語は signals_calendar と**完全に同一**: 色=パターン（v4矢印10色）、形=方向▲▼、pass_all=FALSE は薄表示
- 表示粒度はセル圧迫を避けるため小ドット、詳細はホバー/ドリル内で

## 5. 視覚情報のヒエラルキー（v3 の設計原則）

```
【一覧で見える】   色（スコア×方向） / MFE/MAEバー / シグナルドット / トレードマーク
【ホバーで見える】 その日のサマリー（シグナル内訳・トレード有無・スコア値）
【クリックで見える】評価シート流の詳細カード（トレード詳細・金額もここ）
```

「足すより絞る」（memory: cognitive-load-reduction-is-the-axis）。一覧面は認識、詳細は掘った人だけ。

## 6. 設計確認（あろさん判断待ち）🟡

### Q1: 色相（方向の青/赤）の新ソースはどれにする？

| 案 | ソース | 性格 |
|---|---|---|
| a | **H1 DI spread の日次平均** | スコアの主成分（H1 50%）と同じリズム。一番速い。おぱ推奨 |
| b | D1 DI | 大局方向。速さは中間、ノイズは少ない |
| c | H1 DI を基本に、D1 と不一致の日は彩度を落とす | 方向の確信度まで色で出す（少し複雑） |

### Q2: シート構成は最終的にどうする？

| 案 | 構成 |
|---|---|
| a | **2枚並行**: 日次認識カレンダー（環境の土台）+ 統合評価シート（シグナル×実行の詳細）。役割分担維持 |
| b | 1枚に収斂: v3 で日次カレンダーに全部入るなら、signals_calendar は専用フィルターページ化（将来の PatA×期間フィルタ等の置き場）へ |
