## 2. マニ判断で進めた詳細設計

### 2.1 ADX値 → 濃淡正規化

```python
def adx_normalize(adx_val, lo=5, hi=50):
    return max(0.0, min(1.0, (adx_val - lo) / (hi - lo)))
```

**判断根拠**:
- ADX 値そのままで `hsl(*, *, adx%)` にすると XAUUSD 常用域 (15-45) で差が出にくい
- 5-50 を 0-1 に線形マップして lightness 8%〜30% に乗せる → 常用域で濃淡が明確に出る
- 後工程で z-score / パーセンタイル方式に切り替え可能（候補3つは v0.3 仕様で言及済）

### 2.2 DI spread → 色相

```python
def di_to_hue(di_spread, adx_val):
    if adx_val < 15: return None  # 凪はグレー固定
    norm = max(-1.0, min(1.0, di_spread / 30.0))
    if norm > 0:   hue = 220; sat = 0.3 + 0.5 * norm  # 青系
    else:          hue = 0;   sat = 0.3 + 0.5 * abs(norm)  # 赤系
    return hue, sat
```

**判断根拠**:
- DI spread の常用域は ±30 程度 (実データ確認済)
- 青 (hue=220) と赤 (hue=0) のみ使用。中間色相は使わず、彩度で「中立感」を表現
- 凪 (ADX<15) は色相を完全に抜いてグレーへ (あろさんスケッチ反映)

### 2.3 D1帯 5パターン色

| 局面 | DI方向 | 色 (RGBA) | 意味 |
|---|---|---|---|
| BU | UP | rgba(80,160,255,0.32) | 順張り買いの地合い |
| BU | DOWN | rgba(220,180,80,0.30) | BU期だが DOWN = 警戒 (黄土色で示唆) |
| PD | DOWN | rgba(220,100,120,0.30) | 順張り売りの地合い |
| PD | UP | rgba(180,80,200,0.28) | PD期だが UP = 警戒 (紫色で示唆) |
| 凪 (ADX<18) | - | rgba(120,120,135,0.28) | 環境が動いてない |

**判断根拠**:
- 「BU/PD だけ」ではなく「BU/PD × DI方向」の組み合わせで色を変える方が情報量が多い
- 透明度は v0.3 仕様の「5-15%」より少し濃く (28-32%) に上げた → スケッチで帯がもっと目立っていたため

### 2.4 セルサイズ

- `min-height: 160px` (v0.3 ASCII の比率を再現するため拡大)
- 月単位 4ヶ月分が縦に並ぶ → スクロールで全期間が見渡せる長さに収まる

### 2.5 H1 補助の表示方式

- H1 ADX 値そのまま `H1:25` の形式
- 背景バーは `hsl(210, 30%, lightness)` の単色グラデーション (DI は表示しない)
- v0.3 仕様の「A案 数値+細い背景帯」を採用 (H4 と同サイズ厳禁)

### 2.6 シグナル列の暫定実装

- 現状: トレード発生日 = ▲ (黄色) / 無し = · 表示
- 本来: v4 mq5 シグナル発火ログから「シグナル発火したがエントリーしなかった日」も拾うべき
- → **コー案件で v4 mq5 のログCSV出力対応が必要** (取り逃がし/過剰反応の検出のため)

## 3. 暫定値・既知の不足

### 3.1 環境データの日次粒度

**現状**: `data/weekly_waves.json` (週次) を **日次マップ**。同一週内の月〜金は全て同じ ADX/DI 値となる。

**結果**:
- 「環境が変わるタイミング」 = 週境目で発生
- 同一週内の H4 セル 5日分が同色・同濃度で並ぶ
- 「Mon=18 / Tue=22 / ... / Thu=42」のような日次変動は出ない

**対応方針**:
- **マニ判断**: 日次変動は捏造しない (研究目的汚染回避)
- 「同じ環境が続いた」ことが見えるのも研究上の正解と判断
- 真の日次粒度データは **コー案件で MT5 日次集計 mq5 を実装** してから本実装

**あろさん判断ポイント**:
> 「同週内同値で並ぶ」が許容範囲か / 日次粒度データを優先取得すべきか

### 3.2 シグナル発火データ未整備

**現状**: トレード発生日 = シグナル発火日と仮定。

**問題**:
- 「シグナル発火したがエントリーしなかった日 (取り逃がし)」が拾えない
- 「シグナル発火しなかったがエントリーした日 (過剰反応)」も拾えない

**対応方針**: コー案件 v4 mq5 のシグナル発火ログを CSV 出力 → このスクリプトに読み込み追加

### 3.3 H1 ADX 値の取得元

**現状**: `weekly_waves.json` の `h1_avg_adx` が `null` のことが多い → H4 値を H1 として暫定使用

**問題**:
- H1 ADX が実勢を反映していない
- H1 補助レーンの「シグナル発火と成績照合」用途が機能してない

**対応方針**:
- `ADX_Weekly_Above_v4.csv` から H1 値を `process_wavelog.py` 側で埋め込む処理を追加 (週次レベル)
- 真の日次 H1 ADX は MT5 日次集計待ち

## 5. おぱへの返球 (実装後)

### 5.1 おぱ番人チェック項目

1. **エントリーロジック混入の有無**
   - パターン名 (RISING_DECEL 等) が CSS / JS / 出力に登場していないか → ✅ なし (grep で確認可)
   - 「>35 警告」「ペナルティ」「重み付け」がコードに登場していないか → ✅ なし
   - スコア式が含まれていないか → ✅ 本実装では未導入

2. **階層化原則の保持**
   - H4 が一目で「主役」と認識できるか → セル領域の最大 (`1fr` で残り全部) で達成
   - H1 が H4 と並列に見えていないか → 22px 固定 (H4 の 1/3 程度) で達成
   - D1 がセル内に閉じ込められていないか → セル外の横帯で達成

3. **視認性**
   - 「今日は環境が良かったか」がH4セルだけで一目わかるか → 色相+濃淡+ADX値+DIマークの4軸で表現
   - 「凪の日」と「強い日」のコントラストが明確か → グレー塗り vs 鮮やかな青/赤で対比

4. **3レイヤー分離 (市場環境 / シグナル品質 / 結果)**
   - 環境 = H4 主役 + H1 補助 + D1 帯
   - シグナル = 専用行 (▲ / ·)
   - 結果 = 専用行 (lot / pl / ★)
   - → 物理的に行が分離。混在表示なし

### 5.2 あろさんレビュー請う観点

1. **H4 セル面積感** — 46% 主役感が出ているか
2. **D1 帯の存在感** — 「環境のバックグラウンド」として機能しているか
3. **凪のグレー塗り** — スケッチイメージと合っているか
4. **同週内同値表示** — 「日次粒度の正確データ取得」を急ぐべきか / 「同じ環境連続」が見えればOKか
5. **D1帯の 5パターン色分け** — マニ判断で BU UP=青 / BU DOWN=黄土 / PD DOWN=赤 / PD UP=紫 / 凪=灰 にした。色味の好み調整が必要か
6. **シグナル列の暫定実装** — トレード発生日のみ = ▲ で十分か / シグナル発火ログ取得を急ぐべきか
