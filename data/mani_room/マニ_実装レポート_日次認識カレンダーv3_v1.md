## 2. 実装内容サマリ

### タブ構成（2タブ）
- **タブ1: 全体像** — カレンダーグリッド（月〜土の6列 × 16ヶ月、2025-03〜2026-06）
- **タブ2: 詳細分析** — v2.0 の 上部サマリ / 円グラフ4つ（クリック連動）/ 円グラフ起点ドリルダウン（ソート付）/ フィルター×ピボット×詳細テーブル（ソート付）を欠落なく移設

### タブ1 セルの視覚要素（指示書 §4.1 — この4つだけ）
1. **背景色**: 強度 = 日次合成スコア段階（calc_daily_score 現行式そのまま・段階化テーブル続行）/ 色相 = **H1 DI spread の符号** / H1×H4 DI 不一致 = **彩度一段ダウン（×0.6 の段階、連続でない）**
2. **MFE/MAE バー**: トレード日 = 実測 H4 48h（右=MFE青/左=MAE赤）、非トレード日 = JST14:00 仮想レンジ（淡色 opacity 0.72）。全期間 p95 正規化は v2.0 踏襲
3. **シグナルドット**: v4 発火。signals_calendar v2 と完全同一の視覚言語（PAT_COLORS 10色 / ▲▼ / pass_all=FALSE は opacity 0.3）
4. **トレードマーク**: 金枠 + 方向▲▼（デフォルトON、トグルで「シグナルのみ」即切替）

削除（→ ホバー title / ドロワーへ降格）: H1 ADX 数値・スコア数値・金額・ロット・★・タグ・環境メモ・バー上数値。

### レンジ判定
- **スコアのみ**（`h4_adx46 < 15` 条件を撤去）。グレー2段の色味は v2.0 と同一 rgb 値を踏襲
- ラベル文字「凪」→「レンジ」（凡例・ツールチップ）。ツールチップの「H4 Phase=凪」は ATR 収束の構造ラベル（別概念）なので残置 — nagi-vs-range-distinction 準拠

### ドロワー（セルクリック）
- signals_calendar v2 の実装を流用: シグナルカード（フィルターヒット明細 + MFE/MAE 12→24→36→48h）+ 実トレードカード（同一 mm-steps フォーマット、pips のみ＝金額・ロット非表示、新規理由はデフォルト閉の折りたたみ）
- v3 追加: ドロワー上部に日次環境ミニサマリ（スコア / H1 DI / H4 DI / 不一致表示）

## 5. マニの設計判断（指示書外の決定、レビュー対象）

| # | 判断 | 理由 |
|---|---|---|
| 1 | **方向ソースは h1_di_spread_close**（「日次平均」ではない） | daily_aggregate.csv に H1 DI の mean 列が無く close のみ収録。v1.2 の「DI 方向 = close（確定情報）」設計と整合。mean が必要なら C2 mq5（XAUUSD_Daily_Aggregate_v1.mq5）の改修案件 |
| 2 | **週次フォールバック日（daily CSV 期間外 = 2026-02-12 以前）は H4 DI に退避** | weekly_waves.json に H1 DI が存在しない。該当日はホバーに「方向ソース=H4（週次fallback）」明記 |
| 3 | **D1 帯（週上部の帯）をタブ1 から撤去** | 正書 Q1 確定の論理そのまま:「D1 は因果、このカレンダーは結果の見比べ」。D1 情報はホバーに「（因果・参考）」付きで残置 |
| 4 | **H4 Phase バッジ / 凪離脱警告枠もセルから撤去** | 同上（戦略レイヤー）。ホバーに残置 |
| 5 | **カレンダーを月〜土の6列に拡張** | JST 土曜発火 31 件。5列のままだと完了条件「389 全件」を満たせない（signals_calendar と同判断） |
| 6 | **期間 = 発火∪トレード（2025-03〜）** | 同上、389 全件表示のため。v2.0 のトレード期間のみ（2026-03〜）から拡張 |
| 7 | **不一致時の彩度ダウンは ×0.6 の1段階** | 連続グラデ禁止に従い係数固定。見え方の好みはイテレーションで調整可 |

## 6. メインおぱがレビューすべき箇所

1. **§5-1 の close 採用**: 指示書の文言は「H1 DI spread 日次平均」。close で意図と合っているか、mean 収集（mq5 改修）まで踏むかの判断
2. **§5-3/5-4 の D1帯・Phaseバッジ撤去**: 正書の論理からの演繹だが、明示指示ではない。あろさん実見で「やっぱり帯は欲しい」がありうる
3. **既知の v2.0 由来の挙動（未修正・報告のみ）**: `calc_daily_score` の D1 整合ボーナスは `d1_di_dir in ("UP","DOWN")` 判定だが、daily_aggregate.csv の di_dir 値は `BULL/BEAR/FLAT` → **日次CSV採用日は D1 ボーナス20点が常に0**。「現行式そのまま」指示に従い v3 でも温存した。スコア式を直すなら v2.0/v3 同時の別案件（比較表の整合性が崩れるため）
4. **5/20 のような H1 短期フリップ日の見え方**: 彩度↓で「迷い」表現にしたが、ノイズに見える可能性。実見確認ポイント
5. タブ2 はv2.0 の機能を意図的にそのまま移設（損益サマリ含む）。タブ1 から金額系を消した分、タブ2 との温度差が許容かどうか
