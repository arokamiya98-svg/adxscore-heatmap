## 1. 設計コンセプト

ATR Ratio の「基準線」を **$絶対値** で常時表示する認識ツール。

- **中央値(Ratio 1.0) / 0.7値(収束下限) / 1.2値(拡張)** の **3つの$基準値だけ** を出す。（2026-07-10: §9 の SL/TP 物差し行を1行だけ追加）
- **方向（買い/売り）は焼かない。** 0.7/1.2 での方向は局面（レンジ／BU／DN）で変化する**変数**＝フォワードで判断する領域。値幅の基準線は**不変の明確な指標**、方向は**可変**、をきっぱり分離する（あろさん原則「ATRは値幅、方向と混同しない」）。
- 用途: ① 現場の基準線（今の収束/拡張の$境界を一目で）② 既存 `atr_widget.js` の加熱閾値（手打ち 13.3 等）を、中央値ベースの客観値で**見直す根拠**にする。

### 既存 `atr_widget.js` との住み分け

| | atr_widget（既存） | **ratio_widget（新規）** |
|---|---|---|
| 役割 | 加熱🔴で止める | 収束/拡張の $基準線を出す |
| 表示 | 現在ATR 16/32/H4-8 + 週次回数 | 中央値 / 0.7 / 1.2 の3基準値 |
| 判定 | HIGH閾値（手打ち） | 中央値ベース（自動算出） |

## 2. データ取得（既存 `fetchOHLC` 流用）

- Twelve Data `/time_series`, `symbol=XAU/USD`, `interval=1h`。
- **`outputsize=1600`**（既存の UTC土日フィルタ後 ~1140本を確保。ATR(16)系列から直近960本の中央値を取るため）。
- 1更新 **1 call**（無料枠に余裕）。土日フィルタは既存ロジックをそのまま適用。
- ⚠️ **実機確認**: Twelve 無料で H1 1600本の履歴深度が返るか。足りなければ `outputsize` を詰める／中央値対象本数を「取れた分」に可変フォールバック。

## 3. 計算（心臓部＝BT整合の中央値）

既存 `calculateWilderATR` は**最後の1本**しか返さない。中央値には**各バーのATR系列**が要るので新関数を足す:

```
calculateWilderATRSeries(candles, period) → number[]
  既存 calculateWilderATR と同一ロジック（初期=単純平均→Wilder平滑）で、
  平滑の各ステップの atr を配列にして返す（末尾が最新）。
```

中央値は **BTと寸分同じ定義** で組む（ソース: `data/bt/ATR_WidthSignal_BT_v3bywavelog_gen2.mq5` の `CalcMedian`）:

```
calcMedianBT(series) →
  - atr > 0 の値のみ採用
  - 直近 960本 を対象（末尾960。無ければある分）
  - 有効本数 < 10 は null（無効）
  - 昇順ソート、偶数時は「上側中央値」 tmp[floor(cnt/2)]（下側との平均を取らず上側1点）
```

- `median = calcMedianBT(直近960本のATR16系列)`
- `v07 = median * 0.7`
- `v12 = median * 1.2`

> この定義一致により、カイのBTが炙り出した 0.7/1.2 ゾーンが、そのまま現場の$ラインになる（整合が命）。MT5との数値微差はバーカウント由来で許容（ゾーンのオーダー一致でOK）。
>
> **H4版**（`ratio_widget_h4.js`・別スクリプト・2026-07-02追加）: 同ロジックで `interval=4h` / `period=8` / `median_bars=240`（8週×5×6）/ `outputsize=400`。配色・表示は共通、キャッシュは `ratio_widget_h4_cache.json` で分離。BT側 H4中央値 ≈45.8（2026-06基準）で「H4=33 は Ratio 0.72＝収束下限」を確認済み（局面で中央値が30〜56に振れるため固定値の意味はズレる＝動的表示の要）。

## 4. 表示（3行だけ・方向ラベルなし）

サイズ medium (4x2)。

```
┌────────────────────────────┐
│  XAUUSD ATR基準      07:20 │
│  平均 (1.0)        19.2    │
│  0.7  収束         13.4    │
│  1.2  拡張         23.0    │
└────────────────────────────┘
```

- 3値とも **$ 小数1桁**。計算不能時は "N/A"。
- ラベルは **「収束」「拡張」の値幅状態のみ**。**「買い/売り」は書かない。**
- 色: **カレンダー v0.9 と統一**（値幅局面色・方向色ではない）。平均(中央値)=灰 `#888888` ／ 0.7 収束=紫 `#966ecd`（カレンダー PD=収縮と一致）／ 1.2 拡張=琥珀 `#ebaf37`（BU=拡張と一致）。定数名も `COLOR_CONTRACT`/`COLOR_EXPAND`（局面名）＝良い悪いの価値判断ではない旨をコメントに明記。
- ヘッダー時刻・背景色・padding は既存 `buildWidget` 踏襲。

## 9. SL/TP 物差し行（2026-07-10 追加）

> 発注: おぱ / 承認: あろさん 2026-07-10。v1 で `atr_widget.js` 側に実装したものを本ウィジェット（ATR INFO H1）へ**載せ替え**（atr_widget 側は revert 済み）。**H4版 `ratio_widget_h4.js` には載せない（H1のみ）。**

**位置づけ: 物差しであってルールではない**。執行はバンド位置ベースが主で、これは B1確定式による参考値の常設表示。シグナルではないため閾値・値幅局面色・通知の**対象外**（ラベルは DIM グレー＝基準線3行と視覚的に区別）。

- **表示位置**: 基準線3行（平均 / 0.7 / 1.2）の直下に1行
- **表示例**: `SL/TP        24 / 39`

**式（B1確定・BT世代と同じ物差し）**:
```
ATR_Avg32 = 直近32本の H1 ATR(16) の単純平均（既存 calculateWilderATRSeries の末尾32本平均）
SL_dist   = ATR_Avg32 × 2.0
TP_dist   = SL_dist × 1.6        （RR 1:1.6）
```
- 出典: `data/bt/SPEC_new_BT.md` §2（B1確定）
- **単位: ドル距離をそのまま整数表示**（Math.round。「SL60」「TP100-120」と同じ数え方）
  ⚠️ SPEC_new_BT の「SL_Pips = SL_dist × 100」換算は使わない（別慣行）
- 系列が32本未満（履歴不足）は null → `N/A` 表示
- **キャッシュ対応**: `ratio_widget_cache.json` に `atr_avg32` を追加保存。API失敗時もキャッシュから SL/TP 行が生きる（旧キャッシュにキー無し → `N/A`）
